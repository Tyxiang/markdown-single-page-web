# 项目配置

## 1. 目录和文件的意义

```c
/README.md          // 项目介绍、资源、过程、通用约定

readme.md           // 必备，关于当前目录内容的说明
image/              // 同目录下 markdown 引用的图表
raw/                // 生成图表的原始文件

/design/            // 面向实现的设计
/project/           // 工程文件
/src/               // 源代码
/build/             // 编译相关的资源
/bin/               // 编译后的固件
/test/              // 测试资源
/test/script/       // 测试脚本
/example/           // 应用示例
```

## 2. 项目配置示例

### 2.1. 成片原型项目

```c
/design/            // 面向实现的设计
/design/readme.md   // 设计背景、思路等

/project/           // 工程文件
/project/readme.md  // 环境、配置、相关资源等

/build/             // 编译相关的资源
/build/readme.md    // 编译说明、相关资源等

/bin/               // 编译后的固件
/bin/readme.md      // 烧写说明等

/test/              // 测试资源
/test/script        // 测试脚本
/test/readme.md     // 测试环境、测试方法、测试用例、测试脚本

/README.md          // 项目介绍、资源、过程、通用约定
```

### 2.2. 应用库项目

```c
/design/            // 面向实现的设计
/design/readme.md   // 设计背景、思路等

/src/               // 源代码
/src/readme.md      // 源代码说明

/build/             // 编译相关的资源
/build/readme.md    // 编译说明、相关资源等

/bin/               // 编译后的库
/bin/readme.md      // 编译库的使用说明等

/example/           // 应用示例
/example/readme.md  // 示例说明

/README.md          // 项目介绍、资源、过程、通用约定
```
